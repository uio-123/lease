 # 房屋租赁系统——软件工程大作业项目介绍

 你好！这是一个**软件工程课程大作业**项目，由一个小组（3-5人）合作完成。简单来说，这个项目做了一件事：**开发了一个房屋租赁管理系统**，并且按照软件工程的标准流程，把从"想法"到"代码"再到"报告"的整个过程都记录了下来。

 下面我会像聊天一样，把这个项目从头到尾讲清楚。你不用怕看不懂，我会用最直白的语言解释每个地方。

 ## 这个项目到底在做什么？

 想象一下：你有一套房子想出租，或者你想租房——以前你可能要贴小广告、找中介、打电话约看房，很麻烦。这个系统就是想**用互联网的方式把租房这件事搬到线上**，让房东/管理员可以发布房源、管理合同，让租客可以搜索房间、预约看房、在线签合同。

 它是一个**B/S架构**（Browser/Server，浏览器/服务器模式）的Web应用，你打开浏览器就能用。

 ## 项目文件概览——这个文件夹里都有什么？

 项目文件夹里主要有这几样东西：

 ```
 软件工程大作业要求.md/docx      ← 老师发的作业题目和要求
 房屋租赁系统软件工程大作业报告.docx  ← 最终提交的课程报告（70页+）
 房屋租赁系统软件工程ppt.pptx       ← 汇报用的PPT

 build_visio_diagrams.py           ← Python脚本——自动从Visio导出图片
 fix_report.py                     ← Python脚本——修复报告里的小问题
 project-lease/                    ← 真正的项目代码（Java后端 + 前端）
 ```

 下面我一个一个说。

 ## 1. 作业要求（软件工程大作业要求）

 这是老师发的题目，要求我们按照软件工程的完整流程做一个项目，报告要包含以下部分：

 - **项目概述**——为什么要做这个系统，选什么开发模型（比如瀑布模型还是敏捷开发）
 - **可行性论证**——技术上能不能做？经济上划不划算？法律上有没问题？
 - **需求分析**——系统需要什么功能？用DFD图（数据流图）、用例图、顺序图、状态图来画清楚
 - **系统设计**——总体架构怎么搭？数据库怎么设计？界面长什么样？
 - **编码实现**——真正写代码，并截图展示
 - **测试**——至少做一次白盒测试（看代码内部逻辑）和一次黑盒测试（只看输入输出）
 - **项目管理**——甘特图（时间安排）、配置管理（比如用Git管理代码版本）、产品报价

 简单理解：这门课教的是"怎么科学地做软件开发"，而不是"怎么写代码"。这份要求让你从需求→设计→编码→测试→交付，走一遍完整的软件工程流程。

 ## 2. 项目代码（project-lease/）

 这是真正的核心——一个可以运行的**房屋租赁管理系统**。

 ### 技术栈（用了什么技术）

 | 层面 | 技术 | 说明 |
 |------|------|------|
 | **后端语言** | Java 17 | 最新的长期支持版Java |
 | **后端框架** | Spring Boot 3.0.5 | 当前最流行的Java Web框架 |
 | **数据库** | MySQL 8.0 | 存储用户、房间、合同等数据 |
 | **缓存** | Redis 7 | 加快访问速度，比如存登录状态 |
 | **对象存储** | MinIO | 存图片，比如房间的照片 |
 | **ORM** | MyBatis-Plus 3.5.3 | 让Java代码操作数据库更方便 |
 | **前端（管理端）** | React 18 + Ant Design + Vite | 给管理员用的后台页面 |
 | **前端（用户端）** | React 18 + Ant Design Mobile | H5手机版，租客用手机访问 |
 | **部署** | Docker | 一键启动MySQL、Redis、MinIO等服务 |

 ### 项目结构——代码是怎么组织的？

 这是一个**Maven多模块项目**（Maven是Java的项目构建工具），分成三个模块：

 ```
 lease/
 ├── model/          ← 数据模型层：定义"房间"、"合同"、"用户"这些数据长什么样
 ├── common/         ← 公共模块：一些通用工具，比如JWT登录验证、文件上传、全局异常处理
 └── web/            ← Web层：真正的API接口
     ├── web-admin/  ← 管理员端接口（管理房源、合同、用户）
     └── web-app/    ← 用户端接口（搜索房源、预约看房、签合同）
 ```

 **数据模型（model/）** 定义了这些"东西"：

 - 公寓信息（ApartmentInfo）、房间信息（RoomInfo）——要出租的房子
 - 租约合同（LeaseAgreement）——房东和租客签的合同
 - 用户信息（UserInfo）——租客
 - 系统用户（SystemUser）——管理员
 - 看房预约（ViewAppointment）——约时间去看房
 - 设施信息（FacilityInfo）、费用信息（FeeKey/FeeValue）——房间有什么设施、要交什么费用
 - 还有很多关联表，比如房间和设施的关系、房间和租期的关系

 **管理员端（web-admin/）** 提供了这些功能：

 - 登录验证（验证码+JWT）——管理员登录
 - 公寓管理——增删改查公寓
 - 房间管理——管理每个房间的信息
 - 租约管理——查看和管理租赁合同
 - 预约管理——查看租客的看房预约
 - 用户管理——查看和管理租客信息
 - 系统管理——管理角色、权限、部门、岗位
 - 文件上传——上传房间图片到MinIO

 **用户端（web-app/）** 提供了这些功能：

 - 手机号登录 + 短信验证
 - 浏览/搜索房源
 - 预约看房
 - 在线签租约合同
 - 查看浏览历史

 另外有一个 **lease-app-unused/** 文件夹，里面是已经编译打包好的前端页面（React + TypeScript + Vite），不过目前标记为"未使用"。

 ### 数据库

 数据库叫 `lease`，使用MySQL 8.0。项目里有个 `lease_dump.sql` 文件，是数据库的完整备份，包含所有表结构和示例数据。另外还有个独立的 `system-management.sql`，是系统管理相关的SQL脚本。

 表里包含了：城市/区县信息、公寓/房间信息、设施标签、费用项目、租约期限、付款方式、用户信息、看房预约、浏览历史、系统用户/角色/权限/部门/岗位等。

 ### Docker 一键部署

 项目根目录有一个 `docker-compose.yml` 文件，写好了所有依赖服务的配置。只需要一条命令就能启动：

 - **MySQL**（端口3307）——存数据
 - **Redis**（端口6379）——做缓存
 - **MinIO**（端口9000/9001）——存图片文件

 这些服务在Docker容器里运行，互不干扰，非常适合开发和部署。

 ## 4. 自动化工具——Python脚本

 这个项目最酷的地方在于，写报告这个"体力活"也是用代码自动生成的。

 ### build_lease_report.py（报告生成器）

 这个Python脚本用一个叫 `python-docx` 的库，**自动生成整份Word报告**。它能：

 - 创建格式规范的Word文档（标题、正文、表格、图片）
 - 自动生成各种图表图片（用PIL/Pillow画图库在内存中画出来）
 - 插入数据流图、用例图、架构图、ER图、甘特图等二十多张图片
 - 设置好字体（微软雅黑）、字号、颜色、对齐方式
 - 填充所有章节内容

 ### build_visio_diagrams.py（Visio图导出器）

 这个脚本配合 `房屋租赁系统大作业图稿.vsdx`（一个Visio绘图文件），用Python的 `zipfile` 模块**直接读取Visio文件内部的结构**，把所有绘图页面导出为PNG图片。

 你可能好奇：.vsdx 文件是什么？它本质上是一个压缩包，里面是XML文件描述的图形。这个脚本就是利用这个原理，把每页图都渲染成图片。

 ### fix_report.py（报告修补工）

 这是个救急脚本——当报告生成后发现有小问题（比如文字错了、图片占位符没替换干净），不用重新生成整个报告，用这个脚本做精准修补。

 ## 总结——这个项目教会了你什么？

 如果你是一个计算机专业的学生，看完这个项目，你应该能感受到**软件工程到底在教什么**：

 1. **软件开发不只是写代码**——你要先搞清楚需求（用户想要什么），然后设计（系统怎么搭），再编码（真正写代码），最后测试（确保没bug），每一步都要有文档记录。
 2. **用工具管理复杂度**——Maven管理依赖、Docker部署环境、Git做版本控制、MyBatis操作数据库、Spring Boot搭框架……这些都是工程化工具，让开发更高效。
 3. **文档自动化**——这个项目甚至用Python自动生成了70多页的报告，体现了"能自动化的就不要手动做"的工程思维。
 4. **前后端分离**——后端Java提供API，前端React渲染页面，通过HTTP请求通信，这是现代Web开发的标配架构。

 结论：这不是一个"能用的租房网站"那么简单，它是一个**完整的软件工程实践案例**，从需求分析到代码实现再到文档交付，你能看到一整套软件开发的真实流程。
